The relayer registry supports official frontend discovery and relayer public-key distribution. It does not permission the protocol and it does not create exclusive relay rights.

## What the registry is for

The official frontend needs to know:

- which relayer execution addresses are officially discoverable
- which relayer public keys it may encrypt bundles to

That is what the registry does. It is a discovery layer for the official app. It is not a protocol-level allowlist.

## Registry policy

The official Parly registry supports a maximum of `10` live relayers.

Registration is automatic if validation passes and a slot is available. Inactive relayers are removed from official discovery so slots can reopen without turning the product into a moderation queue or marketplace.

## Register a relayer

Go to [parly.fi/relayer](https://parly.fi/relayer).

You will need:

- the relayer execution address
- the Base64 relayer public key
- a wallet signature proving control of the submitted execution address

Registration succeeds automatically if:

- the registry is open
- the address is valid
- the public key is valid
- the signature is valid
- the relayer is not a duplicate entry

## Registry states

- `verified`: live in official frontend discovery
- `inactive`: previously live, now removed from official discovery because of inactivity, operator withdrawal, manual disable action, or failed health expectations
- `rejected`: failed automatic validation

## Important note

Official listing improves public frontend discovery only. It does not control whether the protocol is open, and it does not stop advanced users, SDK users, or custom frontends from targeting independent relayers directly.
